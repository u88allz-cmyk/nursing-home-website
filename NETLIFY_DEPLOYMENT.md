# 네플리 배포 가이드

## 현재 상황
네플리에서 화면이 보이지 않는 문제는 다음과 같은 이유 때문입니다:

1. **풀스택 프로젝트**: 현재 프로젝트는 백엔드 서버가 포함된 풀스택 애플리케이션입니다
2. **네플리 제한**: 네플리는 정적 사이트 호스팅 전용이므로 Express 서버가 작동하지 않습니다
3. **빌드 오류**: 복잡한 의존성으로 인해 빌드가 완료되지 않고 있습니다

## 해결 방법

### 방법 1: 정적 사이트로 변환 (권장)
상담 문의 기능을 네플리 폼으로 변경하여 완전히 정적 사이트로 만들기

**장점**: 
- 무료 호스팅
- 빠른 로딩
- 간단한 관리

**단점**: 
- 관리자 페이지 제거 필요
- 상담 신청을 이메일로만 받음

### 방법 2: 다른 플랫폼 사용
**Vercel** (추천): 풀스택 지원, 무료 티어
- GitHub 연동 후 Vercel에서 import
- 자동 빌드 및 배포

**Railway**: 풀스택 지원, 무료 티어 (제한적)

## 네플리 배포를 위한 현재 설정

1. **netlify.toml** 파일 생성됨
2. **_redirects** 파일 생성됨 (SPA 라우팅용)
3. **client/index.html**에 네플리 폼 설정 추가됨

## 실제 배포 단계

### GitHub에서 네플리로:
1. GitHub 레포지토리로 이동
2. 네플리 계정으로 로그인
3. "New site from Git" 선택
4. GitHub 레포지토리 연결
5. 빌드 설정:
   - **Build command**: `npm install && npx vite build`
   - **Publish directory**: `dist`
   - **Node version**: 20

### 빌드 오류 해결:
현재 빌드가 오래 걸리는 문제가 있으므로:
1. 불필요한 의존성 제거 필요
2. 또는 Vercel 같은 다른 플랫폼 사용

## 추천 솔루션
**Vercel 사용**을 권장합니다:
- 현재 프로젝트 구조 그대로 배포 가능
- 관리자 페이지 포함 모든 기능 유지
- 무료 티어에서도 충분한 기능 제공
